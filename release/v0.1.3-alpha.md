Lithe 0.1.3-alpha is a portable Windows x86-64 preview that fixes repetitive Vibes discovery and adds a reproducible performance and privacy baseline.

What's new:

- Vibes remembers up to 1,000 seen URLs and treats all pages on a seen domain as already visited.
- Thin DuckDuckGo result sets no longer reintroduce previously seen sites, and exhausted candidate pools never silently clear history.
- Every Vibes query combines the strongest positive interest with a different random exploration category, preventing a narrow two-interest loop.
- New automated coverage verifies exhausted pools, domain-level matching, thin-result behavior, and changing exploration queries.
- `BENCHMARK.md` and a reusable local harness compare Lithe with Chrome, test GPC/DNT/prefetch defaults, explain the resource policy, and document the complete local-first Vibes algorithm.

The July 13, 2026 six-tab baseline is intentionally candid. Lithe used 12 rather than 14 processes and its JavaScript-only portion was 30% faster, but it launched 268% slower, used 53% more peak working-set memory, and consumed 133% more CPU time than Chrome on the test machine. These results establish what must improve; they are not used to claim performance leadership.

This is an unsigned, experimental portable alpha. Extract the entire ZIP before running `lithe.exe`; the executable cannot run by itself. Verify the accompanying SHA-256 checksum. This build does not yet have a production automatic-update channel and should not be treated as a replacement for a browser receiving production security updates.

Vibes classification and the taste profile stay local. Live discovery sends generic category searches to DuckDuckGo and may pre-read up to three candidate pages without cookies, scripts, or referrers.
